# Security Policy

## Reporting a vulnerability

Do not disclose suspected vulnerabilities in a public issue.

Use GitHub's private vulnerability reporting feature when it is enabled. Until
then, contact the repository owner privately through the contact method listed
on the owner's GitHub profile and include:

- the affected component and version or commit;
- reproduction steps or a minimal proof of concept;
- the potential impact;
- any suggested mitigation.

Do not include real credentials, customer content, or personal data in a report.
The project will acknowledge a complete report as soon as practical and will
coordinate disclosure after a fix or mitigation is available.

Only the latest released version and the current `main` branch are supported for
security fixes during the project's initial development stage.
